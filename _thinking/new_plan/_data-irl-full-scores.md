## [S|85] Extrapolating Beyond Suboptimal Demonstrations via IRL from Observations (T-REX) (2019)
- venue=ICML 2019 | cat=offline-irl | paradigm=offline-then-online | data=offline | reward=explicit-reward | cont=yes
- url=arXiv:1904.06387 | code=yes (github.com/hiwonjoon/ICML2019-TREX)
- subscores: rew9 con8 reg9 int8 tra8 cod8 mat9 par9
- idea: Learns an EXPLICIT reward function from RANKED state-only trajectories via a pairwise trajectory-ranking loss (reward of better-ranked trajectory should exceed that of worse), then optimizes a policy with the learned reward. Can extrapolate beyond the best demonstrator because the reward captures intent, not just imitation.
- reason: T-REX is one of the strongest fits for Topic-2: it learns an EXPLICIT scalar reward network from ranked state-only trajectories via a Bradley-Terry pairwise loss, completely offline, and is designed to extrapolate beyond demonstrators rather than clone them. Verified real (arXiv:1904.06387, ICML 2019, official code at github.com/hiwonjoon/ICML2019-TREX). It was demonstrated on continuous MuJoCo with deep nets, so 133D lidar+state into a small MLP reward is well within scope. The reward is intent-capturing and transferable, which directly serves the new-track generalization goal. The only mild catch is that quality of extrapolation hinges on having meaningful, monotonic rankings.
- apply: Auto-rank f1tenth laps by lap time (slow cap10 laps vs faster Dreamer rollouts, or progressively-filtered demos) to create the trajectory ranking T-REX needs, train a reward MLP r(s) (or r(s,a)) on the 133D obs with the pairwise ranking loss entirely offline, then use that reward to label the existing offline dataset and run the current IQL/CORL pipeline; optionally finetune the policy online in f110 with the same frozen reward. Action-free, so the noisy/imperfect action labels in the demos do not contaminate the reward.
- paradigm_note: Reward learning is purely offline (ranked observation trajectories, zero env interaction); the resulting explicit reward is then plugged into any RL stage, which can be offline IQL/CORL on the demos or an online finetune in f110 sim. Clean offline-reward -> online/offline-policy bridge.

## [S|84] Learning Robust Rewards with Adversarial Inverse Reinforcement Learning (AIRL) (2018)
- venue=ICLR 2018 | cat=adversarial-il | paradigm=offline-then-online | data=online | reward=explicit-reward | cont=yes
- url=https://arxiv.org/abs/1710.11248 | code=https://github.com/justinjfu/inverse_rl
- subscores: rew10 con9 reg7 int6 tra10 cod7 mat9 par8
- idea: Adversarial MaxEnt IRL that, unlike GAIL, recovers a reward: structures the GAN discriminator as a reward term g(s) (or g(s,a)) plus a shaping term h(s), so at optimum g recovers the true reward up to a constant and is DISENTANGLED from dynamics, making the learned reward transferable to new dynamics. Optimizes the MaxEnt-IRL likelihood via adversarial training.
- reason: This is the single most on-target method for Topic-2's transferable-reward goal: it explicitly recovers a reward (not just a policy), structured as g(s,a)+h(s) so g is disentangled from dynamics and provably recovers the true reward up to a constant, making it the strongest candidate for transferring to map_easy3. It handles continuous control and deep nets natively. Official code exists (justinjfu/inverse_rl) though rllab-era; the maintained HumanCompatibleAI 'imitation' library has a modern, runnable AIRL. Main cost: vanilla AIRL needs online generator rollouts, so it's not pure-offline, but the sim exists and an offline-pretrain/online-finetune split is straightforward.
- apply: Core of the f1tenth IRL line. Recipe: (1) offline, pretrain a policy with IQL/BC on the fixed cap10 expert demos and warm-start the AIRL discriminator on demos-vs-policy rollouts; (2) online, run the AIRL adversarial loop in f110 (generator = SAC/PPO or your IQL-derived policy) so the reward net g(s,a) sharpens; (3) freeze g, treat it as the learned reward, and either run IQL with it or deploy on map_easy3 to test transfer. For a strictly-offline variant, consider an off-policy/DICE AIRL adaptation (e.g. SaminYeasar/Off_Policy_AIRL) using only logged data.
- paradigm_note: Standard AIRL is on-policy (generator rollouts during reward learning), so the natural fit here is: pretrain the discriminator/reward and a BC/IQL policy on the fixed demos, then finetune the AIRL adversarial loop online in the f110 sim. The disentangled g(s)+shaping(s) reward is exactly what you want to freeze and reuse on a new track.

## [A|79] Coherent Soft Imitation Learning (CSIL) (2023)
- venue=NeurIPS 2023 | cat=frontier | paradigm=offline-then-online | data=hybrid | reward=explicit-reward | cont=yes
- url=https://arxiv.org/abs/2305.16498 | code=yes (github.com/google-deepmind/csil)
- subscores: rew8 con8 reg9 int8 tra6 cod8 mat6 par10
- idea: Shows the behavior-cloned policy can be inverted into a shaped, 'coherent' reward: r(s,a) = alpha*(log q_BC(a|s) - log p(a|s)), i.e. the BC log-likelihood relative to a prior acts as a reward AND defines the critic hypothesis space. This bridges BC and IRL/entropy-regularized RL: start from BC, then optionally fine-tune the policy with this self-derived reward, online or offline.
- reason: CSIL is the cleanest match to the user's favored recipe: BC on the fixed expert data needs no env interaction and yields a closed-form explicit shaped reward you can label transitions with and feed into offline IQL, then optionally finetune online in f110. It is continuous-control native, stable, and low-tuning, with official DeepMind code that has a verified offline mode. The reward is policy-relative (BC-derived), so transfer to map_easy3 hinges on BC generalization, slightly below disentangled-reward methods like ML-IRL/AIRL; otherwise it is a top candidate.
- apply: Top practical pick for the offline->online line. Behavior-clone on the cap10 expert demos, compute the coherent reward r=alpha*(log q_BC - log p) in closed form, relabel the offline dataset and run their existing IQL (offline), then finetune the soft policy online in f110 using the same self-derived reward (use the repo's --offline / online finetune paths). Pair with a more disentangled reward (ML-IRL) if new-track transfer underperforms.
- paradigm_note: Native hybrid: the shaped reward r=alpha*(log q_BC - log p) is derived OFFLINE from a behavior-cloned policy (no sim), then the policy is finetuned with that self-derived reward either offline or ONLINE. The official code exposes an --offline=True flag and supports online finetuning, so it is the textbook offline-pretrain-then-online-finetune method.

- idea: Removes T-REX's need for human rankings: behavior-clone a policy from suboptimal demos, then inject increasing action noise to automatically generate a ranking (more noise = worse), then run T-REX's ranking loss to learn an explicit reward, then optimize a policy with RL.
- reason: D-REX removes T-REX's dependence on human/curated rankings by behavior-cloning a policy from suboptimal demos and injecting increasing action noise to auto-generate a quality ranking, then applying the same explicit-reward ranking loss. Excellent fit when your demos are not pre-ranked. Use that reward to relabel the fixed dataset and run IQL offline, then finetune online. Pick D-REX over T-REX specifically if you don't trust your manual quality ranking; otherwise T-REX is simpler. Could be approximated as strictly-offline by perturbing logged expert actions instead of rolling out.
- paradigm_note: BC + automatic noise-ranking + T-REX ranking loss are all offline once the noisy rollouts exist; the recovered reward then feeds online (or offline IQL) policy optimization. The one online-ish step is generating noise-injected rollouts, which in f110 is cheap or can be approximated offline by perturbing logged actions.

It is natively offline; the recovered reward can then feed any online finetuner, but the method itself does not finetune online.

## [A|76] SMODICE: Versatile Offline Imitation Learning via State Occupancy Matching (2022)

- idea: A fully-offline, regression-based DICE method that minimizes the KL/f-divergence between learner and expert STATE occupancy. Requires only expert observations, no actions, no env interaction.
- reason: SMODICE (verified real, arXiv:2202.02433, ICML 2022, official code at github.com/JasonMa2016/SMODICE) is an excellent regime fit: it needs only a fixed offline dataset and expert OBSERVATIONS, learns a discriminator-based state reward log(d_E/d_O), solves a closed-form-ish dual for importance weights, then does weighted BC. Strongest as a pure-offline imitation baseline against T-REX; weaker if the goal is a reward that transfers cleanly to map_easy3, since occupancy matching anchors to the seen track's state distribution.
- paradigm_note: Fully offline by construction: a discriminator-based state reward plus a DICE Lagrangian dual computed entirely on the fixed offline dataset + expert states, ending in weighted BC. No env interaction is needed or natural; online finetune is possible only by exporting its state reward into a separate RL loop.

## [A|73] Quantifying Differences in Reward Functions (EPIC distance) (2022)

- subscores: rew6 con7 reg9 int7 tra9 cod8 mat7 par6

- reason: EPIC is an evaluation pseudometric, not a reward learner, but it is the right tool to validate Topic-2's generalization hypothesis: it compares two reward functions invariant to potential shaping + positive scaling (the same equivalence class as Ng et al.), is computed offline from sampled transitions, and its EPIC distance bounds the regret of the induced policy even under different transition dynamics. - paradigm_note: EPIC is computed from a FIXED dataset of transitions without any policy training or environment interaction, so it is an offline evaluation/diagnostic tool by construction.
