# 001 — 완주 데이터 없이 2랩 완주 합성 (보고서 §8.1 검증)

> 2026-06-24. 본 보고서 프로젝트와 **별개**로 진행한 추가 실험. append-only.
> 질문: "완주 토대(cap10)도, 1랩 주행 시연조차도 없는 데이터만으로 IQL이 2랩 완주를 합성할 수 있는가?"
> 작성 원칙: 엄밀하되 누구나 읽어서 이해할 수 있게.

---

## 0. 동기
보고서 §8.1의 열린 질문 — D4 성공은 cap10(유일한 2랩 완주 데이터, 30 ep)을 "완주 토대"로 깔고 거기에
고속 충돌 데이터를 스티칭한 것이었다. 그 토대를 **완전히 빼고**, 한 바퀴조차 끝내지 못한 조각들만으로도
2랩 완주가 합성되는지를 본다. 성공하면 "offline RL이 완주 시연 없이 완주를 *발명*에 가깝게 합성한다"는
강한 주장이 되고, 실패하면 "완주 토대가 필요하다"가 답이 된다.

## 1. 데이터 설계 — strict "<1랩"
- **cap10 전면 제외** (2랩 완주 30 ep가 유일한 완주 토대였음).
- cap15·cap20·cap15_rc·cap20_rc에서 **실제 주행거리 < 275.18m(=1랩 미만)** 인 에피소드만 필터.
  - 거리 = Σ |전진속도(state[:,0]×20)| × 0.02s. 스폰 위치·라인통과 아티팩트에 견고한 기준.
  - 원본은 보존, **심볼릭 링크 폴더**(`cap15_rc_sub1lap` 등)로 구성 → 로더가 폴더명에서 cap15/20 파싱 → **조화(harmonization) 정상**.

| 폴더(신규) | 수집 cap | ep | transition | 랩완료(+100) 신호 |
|---|---|---|---|---|
| cap15_rc_sub1lap | 15 | 432 | 101,069 | 5 |
| cap20_rc_sub1lap | 20 | 366 | 79,258 | 3 |
| cap15_sub1lap | 15 | 239 | 78,224 | 0 |
| cap20_sub1lap | 20 | 177 | 98,042 | 3 |
| **합계** | | **1,214** | **356,593** | **11** |

- 로더 적재 기준: **355,379 transition**(raw 356,593 − ep당 1개 정렬 트리밍 1,214), **완주 0 / 충돌 1,214**.
- **핵심 특성 두 가지**:
  1. **완주(2랩)는 물론 1랩 주행조차 0개.** 모든 에피소드가 1랩 미만에서 충돌.
  2. **랩완료 보상(+100) 신호가 11개뿐** → 가치함수는 "랩 완료=좋음"을 사실상 못 본다.
     dense progress + 충돌회피(−50)만으로 2랩을 합성해야 함. 이 실험의 본질적 난관.
- **rc 필수 근거**: rc(센터라인 랜덤 스폰)만이 트랙 전역의 진입 상태를 덮어 폐루프 복구 능력을 학습시킨다
  (D4 ablation에서 입증). 비-rc는 단일 출발점 주행이라 커버리지가 없다.

## 2. 방법 (통제 실험)
- 알고리즘·하이퍼파라미터를 **D4와 100% 동일**하게 고정하고 **데이터 하나만** 바꿈:
  γ=0.999, collision_penalty=−50, β=3.0, iql_τ=0.7, common_v_max=20, lidar 128,
  네트워크 V/twin-Q/Gaussian MLP(2×256), Adam lr 3e-4, batch 256, target 0.005, 상태정규화,
  **600,000 step, 20k마다 ckpt(30개), seed 0**.
- run: `runs/e1_sub1lap` (학습 4,969s ≈ 83분, 정상 종료).
- 평가: 30-ckpt 전수 스윕(seed 0) + 완주 ckpt에 대한 seed 0~3 스윕. 평가 환경 V_MAX=20, 결정론.

## 3. 결과 — **YES, 완주 합성됨**
- **seed 0 스윕: 30개 중 7개 ckpt가 2랩 완주** (220k·280k·300k·320k·380k·460k·560k).
- 최速 단일: **220k = 34.30초** (단, seed 견고성 1/4로 취약).
- **seed 견고성 표 (seed 0~3):**

| ckpt | seed 완주 | 완주 seed별 2랩(초) |
|---|---|---|
| 220k | 1/4 | s0 34.30 |
| 280k | 1/4 | s0 34.72 |
| 300k | 1/4 | s0 34.62 |
| 320k | 1/4 | s0 34.92 |
| 380k | 2/4 | s0 34.80, s2 34.86 |
| 460k | 1/4 | s0 34.52 |
| **560k** | **3/4** | **s0 34.58, s1 34.54, s2 34.62** |

- **가장 견고한 정책 = `checkpoint_560000.pt`, seed 3/4 완주(34.54~34.62초)**, 취약점은 라이다 노이즈.
- 산출물: `runs/e1_sub1lap/sweep_*.json`, `runs/e1_sub1lap/seed_robustness/eval_*_seed*.json` (전량 보존).

## 4. D4(완주 토대 포함)와의 비교
| 항목 | D4 (cap10+15+20+rc) | 본 실험 (<1랩 only, cap10 제외) |
|---|---|---|
| 완주 데이터(2랩) | cap10 30 ep 포함 | **0개** |
| 1랩 주행 시연 | 다수 | **0개** |
| 랩완료(+100) 신호 | 511개 | **11개** |
| transition | 1,023,760 | 355,379 |
| 완주 ckpt(seed0) | 4/30 | **7/30** |
| 최견고 ckpt seed | 3/4 (600k) | **3/4 (560k)** |
| 최견고 2랩 시간 | ~34.3초 | ~34.5초 |

→ **완주 토대를 통째로 빼도 견고성·시간이 D4와 동급**(완주 ckpt 수는 오히려 많음).

## 5. 해석
"모든 상태에서 빠르게+안전하게" 행동하는 **Markov 정책**을 학습하면, 트랙이 루프이므로 랩이 저절로 돌아간다.
필요한 것은 (a) **rc의 전트랙 상태 커버리지**, (b) **dense progress 보상**(γ=0.999로 멀리 전파), (c) IQL의
in-sample 안정성이었다. **완주/expert 시연이나 랩완료 보상은 (적어도 이 트랙에선) 필수가 아니었다.**
→ 보고서 §8.1에 대한 강한 긍정 답.

## 6. 정직한 한계
- seed 3/4(4/4 아님), 취약점은 센서 노이즈. 후반 일부 ckpt는 얇은 데이터(355k)로 불안정(loss 상승).
- **단일 트랙(Oschersleben) + 단일 학습 seed(0)** 결과. 다맵·다seed 일반화는 미검증.
